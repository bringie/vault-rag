## Reindex

After the path-string `REPLACE` was confirmed working via PUT/search smoke, the chunks table was truncated and the indexer re-embedded everything from scratch to ensure no stale rows survived. Reindex driver: `node /scripts/vault-indexer.js` inside `vault-rag-api`.

## Reverting

```bash
# Source / config
git checkout pre-renumber-2026-05-10 -- .

# Postgres
docker exec -i vault-rag-postgres psql -U postgres vault_rag < /tmp/renumber-backup/chunks-pre-renumber.sql
```

## Why now

- Numbering had three orphan prefixes (`04-people`, `05-memory`, `06-prompts`) that were folded into existing categories during the audit. The remaining gap (`02, 03, 05, 06, 08, 09`) made the layout feel arbitrary and confused new contributors.
- After audit, only six "real" categories survived. Contiguous `01..06` is one less thing to remember when capturing notes by hand.
- Historical sessions/tasks predating this layout still resolve via the path-rewrite in postgres.
